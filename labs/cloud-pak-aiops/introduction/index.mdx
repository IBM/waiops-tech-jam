In the Cloud Pak for AIOps labs of the AIOps Jam-in-a-Box series, you will go through several key exercises to learn how to install and configure it.

This is the current lab list:

- **Installation**
  - Install a small size deployment of the Cloud Pak for AIOps
  - Choose between an [OpenShift](../install-lab/1-introduction/index.mdx) based deployment or a [Linux VM](../install-lab-linux/1-introduction/index.mdx) based deployment
  - Perform the installation
  - Calculate custom deployment sizes using the _custom sizing tool_
  - Learn about storage requirements
  - Learn about integration with Lightweight Directory Access Protocol (LDAP)

- [**Topology**](../topology-lab/1-introduction/index.mdx)
  - Create topology data
  - Use rules to merge topology elements
  - Use templates to gather resources into logical groups
  - Define applications and configure business criticality

- [**Alert Correlation**](../alert-lab/1-introduction/index.mdx)
  - Load sample event and alert data using a generic webhook and a Kafka topic
  - Implement scope-based alert correlation
  - Implement temporal alert correlation
  - Implement topological alert correlation

- [**Migration from Netcool**](../migration-from-netcool/1-introduction/index.mdx)
  - Set up an integration into Netcool/OMNIbus
  - Set up an integration into Netcool/Impact
  - Create an event housekeeping automation
  - Create a right-click tool that leverages Netcool/Impact
  - Migrate custom correlation automation from Netcool

- [**Runbook Automation**](../runbook-automation/1-introduction/index.mdx)
  - Create an SSH integration
  - Create runbook actions
  - Create a runbook
  - Create a runbook policy

More labs will be added in the future to explore more features of IBM Cloud Pak for AIOps.
